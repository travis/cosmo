

Custom JCR Node Types - iCalendar


References
==========

http://www.faqs.org/rfcs/rfc2445.html


Node Type Definitions
=========

iCalendar calendars, components, properties, and some property value
types are represented as JCR nodes.

Property values, parameters, and most property value types are
represented as JCR properties and converted to JCR property types
(date, long, double, etc.) where possible, defaulted to String
otherwise.

Some complex property value types are broken down into multiple JCR
properties (eg icalendar:version has two properties,
icalendar:minversion and icalendar:maxversion) instead of being
represented as nodes, for reasons of simplicity.

Inheritance Hierarchy:

  nt:base
  |
  |--icalendar:calendar
  |
  |--icalendar:component
  |  |
  |  |--icalendar:event
  |  |
  |  |...
  |
  |--icalendar:property
  |  |
  |  |--icalendar:textproperty
  |  |  |
  |  |  |--icalendar:description
  |  |  |
  |  |  |...
  |  |
  |  |--icalendar:datetimeproperty
  |  |  |
  |  |  |--icalendar:dtend
  |  |  |
  |  |  |...
  |  |
  |  |--icalendar:prodid
  |  |
  |  |...
  |
  |--icalendar:valuetype
     |
     |--icalendar:period
     |
     |...
